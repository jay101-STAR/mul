# 3周期乘法器设计文档

## 📋 项目概述

这是一个基于 Booth Radix-4 编码的3周期流水线乘法器，将原来的单周期乘法器改造为3周期流水线结构，实现频率提升约2倍。

## 🏗️ 设计架构

### 流水线划分

```
周期1: Booth编码 + 第一层4:2压缩
       ↓ [流水线寄存器1: 620位]
周期2: 第二层4:2压缩 + 第三层4:2压缩
       ↓ [流水线寄存器2: 206位]
周期3: 全加器 + 最终加法
```

### 模块结构

```
mul_3cycle (顶层模块)
├── booth (Booth编码)
├── compress × 7 (4:2压缩器)
│   ├── compress1-4 (第一层，4个并行)
│   ├── compress5-6 (第二层，2个并行)
│   └── compress7 (第三层，1个)
└── full_adder (全加器)
```

## 🔌 接口定义

### 输入信号

| 信号 | 位宽 | 说明 |
|------|------|------|
| `clk` | 1 | 时钟信号 |
| `rst_n` | 1 | 复位信号（低电平有效） |
| `mul_a_i` | 32 | 乘数A |
| `mul_b_i` | 32 | 乘数B |
| `mul_a_sign` | 1 | 乘数A符号控制（1=有符号，0=无符号） |
| `mul_b_sign` | 1 | 乘数B符号控制（1=有符号，0=无符号） |
| `mul_start` | 1 | 启动信号（脉冲） |

### 输出信号

| 信号 | 位宽 | 说明 |
|------|------|------|
| `mul_result` | 64 | 乘法结果 |
| `mul_done` | 1 | 完成信号（脉冲） |
| `mul_busy` | 1 | 忙碌信号 |

## 🔍 状态机

```
IDLE → CYCLE1 → CYCLE2 → CYCLE3 → IDLE
  ↑                                  ↓
  └──────────────────────────────────┘
```

### 状态说明

- **IDLE**: 空闲状态，等待 `mul_start` 信号
- **CYCLE1**: 第一周期，执行 Booth编码 + 第一层压缩
- **CYCLE2**: 第二周期，执行第二、三层压缩
- **CYCLE3**: 第三周期，执行全加器 + 最终加法，输出结果

## 📝 使用示例

### Verilog 代码示例

```verilog
// 实例化3周期乘法器
mul_3cycle mul_inst (
    .clk       (clk),
    .rst_n     (rst_n),
    .mul_a_i   (32'd10),
    .mul_b_i   (32'd20),
    .mul_a_sign(1'b0),      // 无符号
    .mul_b_sign(1'b0),      // 无符号
    .mul_start (mul_start),
    .mul_result(mul_result),
    .mul_done  (mul_done),
    .mul_busy  (mul_busy)
);

// 启动乘法
initial begin
    mul_start = 1'b1;
    @(posedge clk);
    mul_start = 1'b0;

    // 等待完成
    wait(mul_done);
    $display("Result: %d", mul_result);
end
```

## 🔧 集成到CPU

### 方法1：停顿流水线（推荐）

```verilog
// 在ID阶段检测乘法指令
if (is_mul_instruction) begin
    mul_start <= 1'b1;
    stall_pipeline <= 1'b1;  // 停顿2个周期
end

// 等待乘法完成
if (mul_done) begin
    stall_pipeline <= 1'b0;
    // 使用 mul_result
end
```

### 方法2：独立乘法单元

将乘法器作为独立功能单元，与主流水线并行运行。

## 📈 资源使用

### 流水线寄存器

- **寄存器1**: 8×69位 + 1×68位 = 620位
- **寄存器2**: 2×69位 + 1×68位 = 206位
- **总计**: 约826位

### 逻辑资源

- Booth编码器: 1个
- 4:2压缩器: 7个
- 全加器: 1个
- 状态机: 2位状态寄存器

## ⚠️ 注意事项

1. **启动信号**: `mul_start` 必须是单周期脉冲
2. **复位**: 使用异步复位，低电平有效
3. **时序**: 确保时钟周期满足关键路径要求
4. **结果有效**: 只在 `mul_done=1` 时结果有效

## 🐛 调试提示

### 查看状态机

在波形中观察 `state` 信号，应该看到：
```
IDLE(00) → CYCLE1(01) → CYCLE2(10) → CYCLE3(11) → IDLE(00)
```

### 检查流水线寄存器

- 在 CYCLE1 结束时，检查 `stage1_out*` 寄存器
- 在 CYCLE2 结束时，检查 `stage2_out*` 寄存器

### 常见问题

1. **结果不正确**: 检查流水线寄存器是否正确保存数据
2. **mul_done不拉高**: 检查状态机是否正常转移
3. **时序违例**: 增加时钟周期或优化关键路径
